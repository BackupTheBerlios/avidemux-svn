MP3 vbr

1.La base

    Nella maggioranza dei casi, avidemux calcola l'offset e la lunghezza dell'audio con la moltiplicazione dimensione = tempo * bitrate. Ciò funziona per tutti i flussi CBR.

    Non funziona con i flussi VBR in quanti il bitrate non è costante.

2.Temporizzazione audio

Temporizzazione audio 

    Per calcolare l'offset di partenza/dimensione, avidemux deve conoscere come varia il bitrate. Utilizzare il menu Audio source->audio time map.

    Avidemux decomprimerà l'intero flusso crendo poi una mappa di temporizzazione per poter essere in grado di valutarlo. E' necessario avere Mad installato, altrimenti questa opzione sarà inefficace.

    A mappa di temporizzazione costruita, avidemux avrà a disposizione una tabella di conversione tra tempo ed offset per poter dividere o salvare il file avi in VBR.

E se non lo faccio? 

    Il risultato sarà un file avi desincronizzato difettoso (potenzialmente con un audio di durata molto inferiore), anche se lo caricherete e salverete solamente.

    E' un'operazione NECESSARIA per gli mp3 VBR. Punto.

    Il secondo flusso audio nel file avi non può essere VBR.
