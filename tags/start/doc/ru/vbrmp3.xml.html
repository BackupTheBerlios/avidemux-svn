VBR MP3

1.Оступление

    В большинстве случаев avidemux расчитывает смещение и длину аудио по формуле size= time * bitrate. Это применимо для всех CBR потоков.

    Для VBR это неправильно, т.к. битрейт не постоянен.

2.Audio time map

Audio time map(временная карта звука) 

    Чтобы расчитать смещение/размер звуквой дорожки нужно знать, как изменяется битрейт. Для этого используется Audio source->audio time map.

    Avidemux распакует поток и просчитает временную карту изменений. Для работы необходим пакет Mad , иначе ничего не получится.

    Когда карта создана, avidemux сможет корректно обрабатывать(вырезать, сохранять, разбивать) avi с VBR звуком

А если я этого не сделаю ? 

    Вам же хуже, получите рассинхронизированный avi (чаще всего звук будет короче), даже если вы только загрузили и сохранили его.

    Вы ДОЛЖНЫ делать это приработе с VBR mp3. Всегда.

    Второй аудио поток в avi не может быть VBR.
