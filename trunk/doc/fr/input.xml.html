4.Problèmes de codecs

Problème pour décompresser l'image 

    Cela veut dire soit que le codec a renvoyé une erreur, soir qu'Avidemux n'a pas pu trouver le codec vidéo approprié.

    Vous pourrez traiter l'audio et enregistrer le fichier avi malgré cela, mais vous ne pourrez pas traiter la vidéo.

Enregistrer l'audio et la vidéo génère un AVI défectueux Enregistrer l'audio fonctionne 

    Même message pour l'audio. Le codec audio n'a pu être trouvé (pas compilé ou inconnu).

    Vous pouvez enregister et couper l'avi, mais pas traiter l'audio.

Après enregistrement, l'audio est plein de bruit (WMA) 

    Le format WMA a des exigences particulières au niveau du multiplexage. Regardez la valeur blockalign dans la console (généralement 373 ou 768)

    Saisissez cette valeur dans le champ bytes per packet de la fenêtre file -> mux options.

5.Problèmes spécifiques à un format

Mpeg file 

    Avidemux ne peut pas ouvrir de flux mpeg bruts. En revanche, il peut ouvrir des flux mpeg indexés. L'index est un fichier texgte contenant des informations sur la navigation. Il est indispensalbe pour repérer les images avec précision.

    Vous pouvez passer par le menu misc->index mpeg ou ouvrir directement le mpeg. Dans ce cas, vous serez redirigé vers la boîte de dialogue index mpeg dont les champs seront préremplis. Il fois l'index créé, il sera réouvert de façon transparente.

    Si le nom du fichier est quelque chose comme foo1.xxx, Avidemux essaiera d'ajouter foo2.xxx, foo3.xxx. Cela se fait automatiquement, mais il est impératif que le nom du fichier soit de la forme *[0-9].xxx.

    Avidemux peut aussi gérer un (et un seul) flux audio du mpeg. Ce flux est repéré par son identifiant de (sous-)flux.

    Il vaut C0--c7 pour de l'audio mpeg (.mpg, vcd,...), A0--A7 pour de l'audio PCM et 0--7 pour de l'audio AC3.

    Ainsi, pour désigner la seconde piste audio AC3 de votre fichier mpeg, entrez 1.

    Bien sûr, il vous faudra le codec audio adéquat (mad ou a52) pour le pouvoir lire.

Nuppel Video 

    Comme les flux mpeg, un fichier nuppel video n'a pas d'index interne.

    Avidemux le générera automatiquement lors du chargement (l'opération peut prendre plusieurs minutes).

    Avidemux fera de son mieux pour conserver la synchronisation audio/vidéo, mais de temps à autre, un "plop" peut survenir. D'après mon expérience personnelle, cela fonctionne plutôt bien (j'ai fait un essai avec un fichier nuppel_gatos de huit heures qui n'a pas posé de problèmes).

    Si vous êtes passé par le menu (et pas par la ligne de commande), le programme vous demandera au chargement si vous voulez enregistrer un index. Si vous répondez oui, en rechargeant cet index, vous rechargerez le même fichier nuv mais bien plus rapidement.

WMV2 Video 

    Bien qu'Avidemux soit à présent capable d'ouvrir le contenu d'un fichier WMV2 (format de Windows Media Player 7.x/8.x), les données vidéo doivent se trouver dans un conteneur.

    Pour créer ce conteneur, vous pouvez utiliser le paquetage MEncoder en mode copie (par exemple : mencoder [fichier wmv] -oac copy -ovc copy -cache 8192 -ofps 100.0 -o [fichier avi]

    Pour le moment, vous devez également forcer la cadence du fichier de sortie à 100.0 images par seconde (IPS), sans quoi vous perdrez de nombreuses images lorsque vous éditerez votre métrage vidéo réel.

    Une fois votre fichier de sortie créé, vous pouvez le charger et l'éditer dans Avidemux.

Vidéo BMP/ Vidéo Jpeg 

    Avidemux peut également ouvrir un flux de fichiers bmp sans compression (rvb) ou une série de fichiers jpeg.

    Ils doivent être nommés selon le schéma xx0000.bmp xx00001.bmp (à savoir m lettres + n chiffres+.bmp). Pour les jpeg, il suffit de changer l'extension en jpg.
