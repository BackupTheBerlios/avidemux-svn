Installer Avidemux

1.Paquetages nécessaires

Il vous faudra une machine avec une architecture Unix, de préférence x86. Avidemux peut également fonctionner dans ue certaine mesure sur Sparc et Mac

Paquetage Commentaire
Gtk+ et glib 2.x et pkg-config (correctement configuré !)
libmad Il est facultatif, mais faites-moi confiance, vous voudrez l'avoir !
libxml2 Sert aux descriptions des filtres et à ~/.avidemuxrc
nasm (pour x86) Si vous vous intéressez aux VCD/SVCD/... sur x86, vous voudrez l'avoir ! (5 fois plus rapide)

3.Installation

Elle est assez évidente.

./configure

Portez une attention toute particulière aux derniers messages qui s'affichent : ils récapitulent ce qui a été détecté. Pour moi, ils ressemblent à ceci :

Si vous pensez qu'il vous manque quelque chose, vérifiez que vous avez également installé le paquetage -dev.

La commande configure créera également la bibliothèque lavcodec.

Une fois cette commande achevée, faites make puis make install . Un paquetage de traductions (un début d'internationalisation) s'installe également.

4.Problèmes d'installation courants

Echecs lors de la compilation 

La compilation échoue parce qu'il est impossible d'ouvrir gtkxxx. Cela veut généralement dire que pkg-config/gtk+/glib n'est pas installé ou pas configuré.

Elle échoue parce qu'il est impossible de lier jpeg mmx 

Regardez le fichier /usr/lib/liblavjpeg.la (ou dans /usr/local/lib) Si vous trouvez :

Code listing 4.1

		dependency_libs=' -L/usr/src/redhat/BUILD/mjpegtools-1.6.0/jpeg-mmx -ljpeg-mmx'
modifiez-le en

Code listing 4.2

		dependency_libs=''
Merci à Jens pour ce point, qui intéressera beaucoup d'utilisateurs RedHat et Suse.
